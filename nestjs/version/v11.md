# NestJS v11 주요 변경 사항

## 개요

NestJS v11은 2025년 2월에 릴리스되었으며, Fastify v5 지원, Express v5 지원, 새로운 Logger 시스템, 성능 최적화 등 현대적 Node.js 생태계에 맞춘 대규모 업데이트입니다.

## Node.js 최소 버전 상향

- Node.js 16 지원 중단
- **최소 Node.js 18** 이상 필요
- Node.js 20, 22 LTS 공식 지원

## Express v5 지원

- `@nestjs/platform-express`가 **Express v5**를 기본으로 지원
- Express v5 주요 변화:
  - 라우트 파라미터 매칭 규칙 변경(정규식 기반 축소)
  - `res.send()` 동작 일부 변경
  - 비동기 에러 핸들링 개선
- Express v4도 호환 유지

## Fastify v5 지원

- `@nestjs/platform-fastify`가 **Fastify v5**를 기본으로 지원
- Fastify v5 주요 변화:
  - 플러그인 시스템 개선
  - TypeScript 타입 안정화
  - 성능 추가 최적화

## 새로운 Logger 시스템

- 내장 `ConsoleLogger`의 기본 출력 형식을 **JSON 구조적 로깅**으로 변경 가능
- 커스텀 로거 통합이 더욱 간편해짐
- `LoggerService` 인터페이스 확장: 컨텍스트 메타데이터 지원 강화

```ts
const app = await NestFactory.create(AppModule, {
  logger: new ConsoleLogger({
    json: true, // JSON 포맷 출력
    colors: false, // 프로덕션 환경용
  }),
});
```

## Guard, Interceptor, Pipe의 Enhanced DI

- Guard, Interceptor, Pipe에서 **요청 스코프(Request Scope) 지원 개선**
- `@Injectable({ scope: Scope.REQUEST })`를 Guard/Interceptor에서도 안정적으로 사용 가능
- 지연 로딩(Lazy Loading) 모듈과의 통합 강화

## 마이크로서비스 개선

- gRPC 전송 계층 성능 최적화
- NATS JetStream 네이티브 지원
- **`@nestjs/microservices`**에서 커스텀 전송 전략 API 개선
- 이벤트 기반 메시징 패턴 강화

## CLI 및 개발 경험 개선

- `nest new` 프로젝트 생성 시 기본 구조 변경
- **SWC 빌더가 기본값**으로 설정
- HMR(Hot Module Replacement) 설정 간소화
- `nest generate` 커맨드에 새로운 스키매틱 추가

```bash
# v11에서는 SWC가 기본 빌더
nest new my-project
# TypeScript 컴파일러 사용 시 명시적 지정 필요
nest start --builder tsc
```

## 모듈 시스템 개선

- **Dynamic Module** 생성 패턴 간소화
- `ConfigurableModuleBuilder` 개선: 더 직관적인 API
- 순환 의존성 탐지 및 에러 메시지 개선

## 기타 변경 사항

- **Swagger**: `@nestjs/swagger` v8 호환, OpenAPI 3.1 지원
- **GraphQL**: Apollo Server v4 안정화, Mercurius 어댑터 개선
- **WebSocket**: `@nestjs/websockets` 성능 개선 및 Socket.IO v4 안정화
- **Throttler**: `@nestjs/throttler` v6 호환
- **TypeORM / Prisma**: 최신 버전 호환성 테스트 강화
- **Deprecation 제거**: v9, v10에서 deprecated된 API 최종 제거

## 마이그레이션 가이드

1. Node.js 18 이상으로 업그레이드
2. `@nestjs/common`, `@nestjs/core`, `@nestjs/platform-*` 등 핵심 패키지를 v11으로 업데이트
3. Express v5 또는 Fastify v5 호환성 확인
4. deprecated API 사용 부분 수정
5. Logger 커스텀 구현체가 있다면 새 인터페이스에 맞춰 업데이트
6. 마이크로서비스 전송 전략 변경 사항 확인

```bash
# 핵심 패키지 일괄 업데이트
npm i @nestjs/common@11 @nestjs/core@11 @nestjs/platform-express@11
npm i -D @nestjs/cli@11 @nestjs/schematics@11 @nestjs/testing@11
```
