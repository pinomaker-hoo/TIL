# NestJS v10 주요 변경 사항

## 개요

NestJS v10은 2023년 7월에 릴리스되었으며, Node.js 최소 버전 상향, SWC 빌더 공식 지원, 의존성 업그레이드 등 플랫폼 현대화에 초점을 맞춘 메이저 업데이트입니다.

## Node.js 최소 버전 상향

- Node.js 12, 14 지원 중단
- **최소 Node.js 16** 이상 필요
- Node.js 18, 20 LTS 공식 지원

## SWC (Speedy Web Compiler) 공식 지원

- `@swc/cli`와 `@swc/core`를 활용한 빌드 공식 지원
- TypeScript 컴파일 속도 약 **20배 향상**
- Nest CLI에서 `--builder swc` 옵션으로 사용 가능

```bash
nest start --builder swc
# 또는 nest-cli.json에서 설정
```

```json
// nest-cli.json
{
  "compilerOptions": {
    "builder": "swc"
  }
}
```

## 의존성 업그레이드

- **RxJS 7.8.x** 기본 사용
- **TypeScript 4.9 ~ 5.x** 지원
- `@nestjs/platform-express`: Express 4.18.x 기반
- `@nestjs/platform-fastify`: Fastify v4 기반
- `class-validator`, `class-transformer` 최신 호환

## Redis 기반 Microservice 변경

- `ClientRedis` / `ServerRedis` 내장 전략 제거
- `@nestjs/microservices`에서 Redis 전략을 별도 패키지 `ioredis`로 이전
- Redis 사용 시 커스텀 전략 또는 공식 어댑터 사용 권장

## CacheModule 분리

- `@nestjs/common`에서 `CacheModule` 제거
- **`@nestjs/cache-manager`** 별도 패키지로 분리
- `cache-manager` v5 기반으로 업그레이드

```bash
npm i @nestjs/cache-manager cache-manager
```

```ts
import { CacheModule } from '@nestjs/cache-manager';

@Module({
  imports: [CacheModule.register()],
})
export class AppModule {}
```

## Logger 개선

- `ConsoleLogger` 및 `Logger` 클래스에 `fatal` 로그 레벨 추가
- 로그 메서드 시그니처 개선: `log(message, context)` → `log(message, ...optionalParams)`

## 기타 변경 사항

- **GraphQL**: `@nestjs/graphql` v11과 호환, Apollo Server v4 지원
- **Swagger**: `@nestjs/swagger` v7과 호환
- **Testing**: `@nestjs/testing`에서 `TestingModule`의 `overrideProvider` 체이닝 개선
- **Middleware**: 미들웨어 등록 시 와일드카드 라우팅 동작 명확화
- `HttpModule`: `@nestjs/axios`로 완전 이전 (v9에서 시작된 변경 완료)

## 마이그레이션 가이드

1. Node.js 16 이상으로 업그레이드
2. `@nestjs/common`, `@nestjs/core`, `@nestjs/platform-*` 등 핵심 패키지를 v10으로 업데이트
3. `CacheModule` 사용 시 `@nestjs/cache-manager`로 전환
4. Redis 마이크로서비스 전략을 커스텀 구현으로 변경
5. 더 이상 사용하지 않는(deprecated) API 제거
6. RxJS 7.x 호환성 확인

```bash
# 핵심 패키지 일괄 업데이트
npm i @nestjs/common@10 @nestjs/core@10 @nestjs/platform-express@10
npm i -D @nestjs/cli@10 @nestjs/schematics@10 @nestjs/testing@10
```
